## Part I: Understanding Maieutic

### 1.1 What is Maieutic?

Maieutic is a **meta-framework** â€” a tool designed to create other tools. Named after the Socratic method of "midwifery" (drawing forth knowledge through dialogue), it provides:

- **Governance:** Strict rules for naming, documentation, and coding.
- **Workflows:** 15 slash-command triggers for AI-assisted development.
- **Personas:** 8 distinct AI personality modes for different cognitive tasks.
- **Self-Propagation:** The ability to initialize itself into new projects.

### 1.2 Core Philosophy: Local Sovereignty

Maieutic enforces a **Local-First** philosophy:
- **Database:** SQLite (no external dependencies)
- **Network:** Offline-capable, no cloud requirements
- **Data Owner:** You are the sole owner of all data

## Part II: Deployment Methods

There are **two deployment scenarios** for Maieutic:

### 2.1 Method A: Cloning the Parent Repository (For Development)

Use this method if you intend to contribute to or extend the Maieutic framework itself.

```powershell
# Step 1: Clone the repository
git clone https://github.com/[YOUR_ORG]/maieutic.git
cd maieutic

# Step 2: Create a Python virtual environment
python -m venv venv

# Step 3: Activate the virtual environment (Windows)
venv\Scripts\activate

# Step 4: Install dependencies
pip install -r requirements.txt

# Step 5: Verify installation
framework\run\help.bat
```

## Part IV: First Workflow Execution

Once Maieutic is initialized, you interact with it through **slash commands** given to your AI assistant.

### 4.1 The Workflow Lifecycle

```mermaid
graph LR
    A[/plan] --> B[Active Plan Created]
    B --> C[/build]
    C --> D[Implementation]
    D --> E[/audit]
    E --> F{Pass?}
    F -->|Yes| G[/update]
    F -->|No| C
    G --> H[Plan Archived]
```

### 4.2 Essential Commands

| Command | When to Use | What It Does |
|---------|-------------|--------------|
| `/plan [TASK]` | Starting new work | Creates a numbered plan in `docs/dev-plans/` |
| `/build` | After plan is approved | Implements the `active_plan.md` |
| `/audit` | After implementation | Scans for regressions and governance violations |
| `/critique [TARGET]` | To identify bloat | Harsh review of code/architecture |
| `/update` | After plan completion | Archives plan, resets workspace |

### 4.3 Example: Creating Your First Plan


## Part V: Governance Rules

Maieutic enforces strict governance through the `manifest.json` file. Violations are flagged by The Auditor persona.

### 5.1 Naming Conventions

| Type | Format | Example |
|------|--------|---------|
| Directories | `lowercase-hyphen-separated` | `my-feature/` |
| System Files | `ALL_CAPS` | `ISSUES.md` |
| Regular Files | `lowercase_underscore_separated` | `my_module.py` |
| Reports | `TYPE-YYYYMMDD-HHMMSS.md` | `AUDIT-20251219-143000.md` |

### 5.2 Operational Rules

| Rule ID | Description |
|---------|-------------|
| `RUN_001` | Every `.bat` file in `framework/run/` MUST be documented in `help.bat`. |
| `RUN_002` | An archive script MUST exist to keep only the latest report in `docs/reports/`. |
| `PERSONA_001` | Personas are IMMUTABLE. They cannot be removed or consolidated. |

## Part VI: Troubleshooting

### 6.1 "manifest.json already exists"

**Cause:** You are trying to run `maieutic-init.bat` in a directory that already has Maieutic initialized.

**Solution:** Initialize in a fresh, empty directory.

---

### 6.2 "Template directory not found"

**Cause:** The `template/` folder is not alongside `maieutic-init.bat`.

**Solution:** Ensure both the script and the template folder are in the same directory.

---

### 6.3 Placeholders remain in `manifest.json`

**Cause:** PowerShell string replacement failed (special characters in input).

**Solution:** Manually edit `manifest.json` to replace `[PROJECT_NAME]`, `[VERSION]`, `[AGENT_IDENTITY]`, and `[PROJECT_GOAL]`.
